艺考培训班教务系统 — 项目需求文档
============================================================

文档用途：辅助 AI 编程工具（Cursor 等）进行代码生成与功能开发，后续可在此基础上补充细节与注意事项。

系统定位：
  - WEB 端：面向教师 / 教务人员，适合大屏数据查看与批量操作
  - 微信小程序端：轻量化移动应用，面向学生与家长，强调即时触达与便捷查看

------------------------------------------------------------

一、技术栈建议（待确认）
------------------------------------------------------------

  1. 前端 WEB：Vue 3 + Vite + Element Plus（可替换为 React + Ant Design）
  2. 小程序：微信原生 / Taro / uni-app（支持多端时选 Taro / uni-app）
  3. 后端：FastAPI (Python)，RESTful API
  4. 数据库：MySQL + Redis（结构化数据 + 缓存）
  5. 文件存储：腾讯云 COS / 本地 MinIO（视频资源存储）
  6. 认证：JWT + 微信 OAuth（小程序用微信授权）


二、角色体系
------------------------------------------------------------

系统共包含四类角色，权限由后台配置：

  1. 教务（admin）    — 使用端：WEB 端，最高管理权限
  2. 教师（teacher）  — 使用端：WEB 端 + 小程序，班级教学管理
  3. 学生（student）  — 使用端：小程序，查看成绩、视频、请假
  4. 家长（parent）   — 使用端：小程序，查看子女信息


三、WEB 端功能模块
------------------------------------------------------------

3.1 数据仪表盘
    模块说明：首页概览，根据登录角色展示不同维度的统计数据。

    3.1.1 功能点
          a. 教务角色展示：教师人数、学生人数、视频资源数、艺考咨询信息数
          b. 教师角色展示：学生人数、家长人数、视频资源数、艺考咨询信息数

    3.1.2 开发要点
          a. 根据登录 Token 中的角色字段，动态渲染不同统计卡片
          b. 数据来源：各业务表聚合查询，建议使用缓存降低数据库压力
          c. 可使用 ECharts / Chart.js 展示趋势图（后续扩展）


3.2 人员信息管理
    模块说明：管理系统内所有用户的基础信息，支持批量操作。

    子模块：人员管理

    3.2.1 功能点
          a. 人员基本信息编辑：修改姓名、性别、联系方式、头像等基础字段
          b. 账号启用/禁用：控制用户是否可以登录系统
          c. 年级/班级分组管理：将学生归属到指定年级和班级

    3.2.2 开发要点
          a. 用户表需包含：user_id、role、name、gender、avatar、phone、is_active、grade_id、class_id
          b. 年级/班级为独立字典表，与用户多对多关联（学生）或一对多（教师带班）
          c. 账号启用/禁用需同步使 JWT Token 失效（黑名单机制或短期 Token）


3.3 视频管理
    模块说明：管理学生上传的视频资源，包含审核与评论功能。

    子模块一：视频审核

    3.3.1 功能点
          a. 查看视频：列表展示所有视频，支持按学生/班级/上传时间筛选
          b. 编辑视频：修改视频标题、标签、所属科目
          c. 删除视频：物理/软删除视频资源

    子模块二：教学评论

    3.3.2 功能点
          a. 点评学生上传视频：教师对指定视频提交文字评语，关联到视频记录

    3.3.3 开发要点
          a. 视频存储建议使用对象存储（COS/MinIO），数据库只存储视频 URL 与元数据
          b. 审核状态字段：pending（待审核）/ approved（通过）/ rejected（驳回）
          c. 评论是否支持富文本（待确认）
          d. 视频标签来自字典管理模块


3.4 成绩管理
    模块说明：录入和维护学生的学科成绩。

    子模块：成绩录入

    3.4.1 功能点
          a. 批量导入：通过 Excel 模板批量上传成绩数据
          b. 编辑/修改成绩：单条记录的增删改操作

    3.4.2 开发要点
          a. 支持 .xlsx 文件上传，使用 openpyxl 或 pandas 解析
          b. 成绩科目维度：练声、鉴赏、乐理、写作、理论、大考视唱、大考声乐、大考器乐、大考总成、大考排名
             【待确认】大考系列是否与日常科目分开独立存储，还是同一张表加类型字段区分
          c. 成绩表关键字段：score_id、student_id、subject、score、exam_date、teacher_id、remark


3.5 字典管理
    模块说明：维护系统基础数据字典，供其他模块调用。

    子模块：系统字典管理

    3.5.1 功能点
          a. 视频标签管理：增删改查视频分类标签
          b. 年级/班级管理：维护年级与班级的层级关系

    3.5.2 开发要点
          a. 字典表设计：dict_type（类型标识）+ dict_value（值）+ dict_label（显示名）+ sort（排序）
          b. 年级/班级建议独立两张表，班级通过外键关联年级


3.6 权限配置
    模块说明：管理系统内角色与功能菜单的访问权限。

    子模块：基础权限配置

    3.6.1 功能点
          a. 角色权限分配：为教务、教师、学生、家长四个角色分配可访问的功能模块

    3.6.2 开发要点
          a. 当前阶段暂定四个固定角色：admin（教务）、teacher（教师）、student（学生）、parent（家长）
          b. 权限粒度目前为菜单/功能级别，暂不做字段级权限
          c. 建议采用 RBAC 模型：用户 → 角色 → 权限
          d. 后端使用中间件/装饰器校验 Token 中的角色


四、微信小程序端功能模块
------------------------------------------------------------

4.1 教师端

    子模块一：信息授权 & 消息录入

    4.1.1 功能点
          a. 微信授权登录：获取微信 OpenID，绑定系统账号
          b. 个人信息录入：录入教师岗位相关信息

    4.1.2 教师必填信息（当前阶段）
          姓名、性别、头像、联系方式
          【待扩展】职务、所教科目、带班信息等

    子模块二：成绩查询

    4.1.3 功能点
          a. 查询学生历史成绩：按班级/学生维度查询成绩记录
          b. 排名查看：查看学生在班级/全体中的排名
          c. 趋势分析：查看学生成绩随时间的变化趋势


4.2 学生端

    子模块一：信息授权 & 消息录入

    4.2.1 功能点
          a. 微信授权登录：获取微信 OpenID，绑定系统账号
          b. 个人信息录入：录入学生必要信息

    4.2.2 学生必填信息（当前阶段）
          姓名、头像、性别、联系方式、班级、宿舍、声乐教师、器乐教师、班主任姓名

    子模块二：视频模块

    4.2.3 功能点
          a. 查看视频：浏览已审核通过的视频资源
          b. 上传视频：学生上传个人练习/考试视频

    子模块三：成绩查询

    4.2.4 功能点
          a. 历史成绩查看：查看各科目历史得分记录
          b. 排名波动变化：展示排名在时间轴上的变化
          c. 成绩趋势曲线：以折线图形式展示成绩走势
          d. 教师点评查看：查看教师对视频的点评内容
          e. 器乐/声乐考试视频：查看对应科目的考试视频

    4.2.5 成绩科目维度
          日常科目：练声、鉴赏、乐理、写作、理论
          大考科目：大考视唱、大考声乐、大考器乐、大考总成、大考排名
          【待确认】大考排名是否需要与其他大考科目成绩区分展示逻辑

    子模块四：在校表现

    4.2.6 功能点
          a. 请假流程：学生发起请假申请，含请假类型、时间、原因
          b. 请假/违纪记录查看：查看历史请假次数、违纪次数及详情

    4.2.7 请假流程设计（建议）
          学生提交申请 → 班主任审核 → 教务审批（可选） → 通知家长（可选）

    子模块五：个人中心

    4.2.8 功能点
          a. 查看个人信息：展示当前账号的基础信息
          b. 编辑个人信息：修改头像、联系方式等可编辑字段


4.3 家长端

    子模块一：信息授权 & 消息录入

    4.3.1 功能点
          a. 微信授权登录：获取微信 OpenID，绑定系统账号
          b. 个人信息录入：录入家长基本信息，并关联子女账号

    4.3.2 家长必填信息（当前阶段）
          姓名、性别、头像、联系方式
          【待确认】家长与学生的关联方式（邀请码 / 手机号绑定 / 后台手动关联）

    子模块二：个人信息

    4.3.3 功能点
          a. 查看个人信息：展示当前家长账号信息
          b. 编辑个人信息：修改可编辑字段

    子模块三：成绩查看

    4.3.4 功能点
          a. 子女历史成绩：查看绑定子女的各科成绩记录
          b. 排名变化：查看子女排名趋势
          c. 变化趋势信息：成绩趋势图表展示

    子模块四：视频查看

    4.3.5 功能点
          a. 查看子女相关视频：浏览子女上传的视频
          b. 教师评论查看：查看教师对子女视频的点评


五、数据库核心表设计（草图）
------------------------------------------------------------

  1.  users               — 用户基础表（含角色、状态、微信 OpenID）
  2.  grades              — 年级表
  3.  classes             — 班级表（关联年级）
  4.  teacher_profiles    — 教师扩展信息
  5.  student_profiles    — 学生扩展信息（含班级、宿舍、关联教师）
  6.  parent_profiles     — 家长扩展信息
  7.  parent_student      — 家长-学生关联表（多对多）
  8.  videos              — 视频表（含状态、标签、上传者）
  9.  video_comments      — 视频点评表
  10. scores              — 成绩表（含科目、考试类型、日期）
  11. leave_requests      — 请假申请表（含状态流转）
  12. discipline_records  — 违纪记录表
  13. dict_items          — 字典数据表
  14. role_permissions    — 角色权限配置表


六、待确认事项清单
------------------------------------------------------------

开发前需逐一确认，避免返工：

  1. 大考排名 是否作为独立科目存储，还是从大考总成自动计算？
  2. 家长与学生绑定方式：邀请码、手机号匹配，还是后台手动关联？
  3. 请假审核流程：是否需要多级审批（班主任 → 教务）？是否通知家长？
  4. 视频点评：是否支持富文本？是否支持语音点评？
  5. 成绩是否区分学期：同一学生同一科目是否按学期归档？
  6. 通知推送：是否需要微信订阅消息推送（成绩发布、审批结果等）？
  7. 多子女支持：家长端是否支持绑定多个子女？
  8. 视频大小与格式限制：上传的约束条件？
  9. 数据权限隔离：教师是否只能看到自己班级的数据，还是全部学生？


七、开发优先级建议
------------------------------------------------------------

  P0 核心
      1. 用户认证（登录/注册/权限）— 所有功能的基础
      2. 人员信息管理 + 字典管理   — 数据基础

  P1 主功能
      3. 成绩录入 + 成绩查询（WEB + 小程序）— 核心业务
      4. 视频上传 + 审核 + 点评              — 核心业务

  P2 次功能
      5. 请假/在校表现模块 — 辅助管理
      6. 数据仪表盘        — 数据展示

  P3 扩展
      7. 权限配置 UI — 初期可硬编码
      8. 通知推送    — 后期接入微信订阅消息
