# 艺考培训班 — 阿里云视频方案二：CDN 流畅型

> 云服务商：阿里云 | 预算：~1.0 ~ 1.4 万/年 | 适用：300 名学生、6000 条视频、10 个月存储周期、追求播放体验

---

## 一、需求确认

| 约束 | 说明 |
|------|------|
| 云服务商 | 阿里云 |
| 学生规模 | 300 人 |
| 视频总量 | 300 人 × 20 条 = **6,000 条** |
| 单条时长 | 约 5 分钟 |
| 录制设备 | iPhone（4K HEVC 为主，部分 Android/其他设备） |
| 视频清晰度 | 双码率：1080p 主码率 + 720p 副码率，支持切换 |
| 存储周期 | **10 个月**（培训周期内保留，过期后归档或清理） |
| 总预算 | **≤ 20,000 元/年** |
| 体验要求 | 上传流畅 + **全国播放流畅** |

## 二、视频体量分析

### 2.1 原始视频体量

| 项目 | 计算 | 结果 |
|------|------|------|
| 单条原始大小 | iPhone 4K HEVC，5 分钟 | ~1.5 GB |
| 原始总量 | 6,000 条 × 1.5 GB | **~9 TB** |

> 原始视频不会直接上传，客户端先压缩转码，大幅减少体积。

### 2.2 压缩后视频体量

| 项目 | 计算 | 结果 |
|------|------|------|
| 1080p H.264 主码率 | 约 150 MB/条 | 6,000 × 150 MB = **900 GB** |
| 720p H.264 副码率 | 约 50 MB/条 | 6,000 × 50 MB = **300 GB** |
| 双码率总存储峰值 | 900 GB + 300 GB | **~1.2 TB** |
| 月均新增（10 个月均匀上传）| 600 条/月 × 200 MB | **~120 GB/月** |

### 2.3 播放流量估算

| 场景 | 计算方式 | 月均流量 |
|------|----------|----------|
| 老师查看点评 | 30 老师 × 200 条 × 150 MB × 50% 实际观看 | ~450 GB |
| 学生回看 | 300 人 × 5 条 × 150 MB | ~220 GB |
| 家长回看 | 200 家长 × 2 条 × 150 MB | ~60 GB |
| **合计（保守估）** | | **~700 GB/月** |

> 流量是成本最大变量。高峰期（考试前后）可能翻倍，CDN 流量包比按量更划算。

## 三、双码率实现方案

### 3.1 推荐架构：客户端单码率 + 服务端异步转码 + CDN 分发

```
客户端录制（iPhone 4K HEVC / Android 等任意格式）
        ↓
客户端压缩 → 统一输出 1080p H.264/MP4（~150MB/5分钟）
        ↓
分片上传到阿里云 OSS（/uploads/raw/ 目录）
        ↓
服务端触发异步转码队列（FFmpeg）
        ├── 1080p 原文件 → 移到 /videos/1080p/
        └── 720p 转码   → 生成 /videos/720p/
        ↓
CDN 分发两个地址：
        ├── cdn.xxx.com/videos/1080p/xxx.mp4（WiFi/5G 默认）
        └── cdn.xxx.com/videos/720p/xxx.mp4（4G/弱网自动降级）
```

### 3.2 转码资源选择

| 转码方式 | 优点 | 缺点 | 费用估算 |
|----------|------|------|----------|
| ECS 本机 FFmpeg | 成本低，配置简单 | 占用 ECS CPU，转码时服务可能卡顿 | 0 元（含在 ECS 中） |
| 阿里云函数计算 FC | 按需付费，不占用 ECS | 需要学习 Serverless 架构 | ~500 ~ 1,500 元/年 |
| 阿里云媒体处理 MPS | 稳定可靠，自动触发 | 费用较高 | ~600 ~ 1,000 元（一次性） |
| 阿里云视频点播 VOD | 一站式，自动转码+CDN | 费用高，超出预算 | ~5,000+ 元（不推荐） |

> 本方案推荐：**ECS 本机 FFmpeg 队列转码**，在凌晨闲时批量处理，白天不占用 CPU。6000 条视频约需 50-100 小时转码时间，可分批次在夜间完成。

## 四、10 个月存储周期策略

### 4.1 生命周期自动管理

```
OSS 生命周期策略（自动执行，无需人工干预）：

  上传后 0-30 天     → 标准存储（频繁访问，0.12 元/GB/月）
  30-90 天          → 低频访问存储（0.08 元/GB/月，降 33%）
  90-300 天（10个月）→ 归档存储（0.033 元/GB/月，降 72%）
  300 天后           → 删除 或 转冷归档（0.015 元/GB/月，降 87%）
```

### 4.2 成本对比（生命周期 vs 不分级）

| 策略 | 年存储费用（1.2TB 峰值）| 节省 |
|------|------------------------|------|
| 不分级（全标准存储）| 1,200 GB × 0.12 × 12 = **~1,728 元/年** | 基准 |
| 分级管理（推荐）| 平均约 **~700 元/年** | **省 ~60%** |

### 4.3 双码率存储优化

```
存储策略：
  最近 1 个月视频（活跃期）：保留 1080p + 720p（双码率）
  1-3 个月视频（温数据）：只保留 720p（省 50% 空间）
  3-10 个月视频（冷数据）：只保留 720p + 归档存储
  10 个月后：自动删除或转冷归档（根据业务需求）
```

## 五、配置清单与成本

### 5.1 资源清单

| 资源 | 产品/规格 | 月费 | 年费 |
|------|----------|------|------|
| 应用服务器 | ECS **4核8G**（ecs.g7.large），带宽 **3M** | ~300 元 | **~3,600 元**（年付） |
| 数据库 | RDS MySQL **2核4G** 基础版 | ~200 元 | **~2,400 元**（年付） |
| 缓存 | Redis 社区版 **1G** | ~70 元 | **~840 元** |
| 对象存储 OSS | 标准存储 + 生命周期管理 | ~60 元 | **~720 元** |
| CDN 加速 | **1TB 流量包/月**（中国大陆）| ~180 元 | **~2,160 元** |
| CDN HTTPS 请求 | 约 100 万次/月 | ~5 元 | **~60 元** |
| OSS 回源流量 | CDN 未命中回源，约 10% | ~15 元 | **~180 元** |
| 域名 + SSL | 万网域名 + 免费 SSL | ~25 元 | **~300 元** |
| 预留弹性/缓冲 | 突发流量、流量超预期 | - | **~1,500 元** |
| **合计** | | **~855 元/月** | **~11,760 元/年** |

### 5.2 费用区间预估

| 场景 | 年费 | 说明 |
|------|------|------|
| 保守估（流量偏低）| **~9,000 ~ 10,000 元** | 月均流量 < 500GB |
| 标准估（正常运营）| **~11,000 ~ 13,000 元** | 月均流量 ~700GB |
| 偏高估（流量较大）| **~14,000 ~ 16,000 元** | 月均流量 ~1TB，需加购流量包 |

> 所有场景均在 **2 万预算内**，方案二留有余量 ~4,000 ~ 11,000 元。

### 5.3 为什么 CDN 方案反而可能更便宜

| 对比项 | OSS 直连（方案一）| CDN 加速（方案二）|
|--------|------------------|------------------|
| 流量单价 | 0.25 ~ 0.50 元/GB | **0.15 ~ 0.24 元/GB**（流量包）|
| 月均 700GB 流量费 | ~280 元 | ~180 元 |
| 年流量费差异 | 基准 | **省 ~1,200 元/年** |

> CDN 流量单价低于 OSS 外网流出，播放流量越大，CDN 方案越划算。

## 六、CDN 配置参数

### 6.1 基础配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 缓存策略 | MP4 文件缓存 7 天 | 热点视频预加载到边缘节点 |
| 回源方式 | OSS 域名回源 | 内网传输免费 |
| HTTPS | 强制 HTTPS | 防止劫持 |
| 防盗链 | 启用 Referer 防盗链 + URL 鉴权 | 防止盗刷流量 |
| 视频拖拽 | 开启 MP4 伪切片 | 支持拖拽播放 |
| 预热策略 | 考试前 1 天预热热门视频 | 减少回源，提升首屏速度 |

### 6.2 CDN 播放 URL 示例

```
原始 OSS URL（不要直接暴露给用户）：
  https://your-bucket.oss-cn-guangzhou.aliyuncs.com/videos/1080p/xxx.mp4

CDN 加速 URL（给用户使用）：
  https://cdn.yourdomain.com/videos/1080p/xxx.mp4

带鉴权的 CDN URL（防盗链）：
  https://cdn.yourdomain.com/videos/1080p/xxx.mp4?auth_key=xxx&timestamp=xxx
```

## 七、关键省钱技巧

### 7.1 年付折扣（必做）

阿里云 ECS/RDS **包年包月比按量付费便宜 40-60%**：

| 产品 | 按量付费/月 | 包年包月/月 | 年省 |
|------|------------|------------|------|
| ECS 4核8G | ~550 元 | ~300 元 | **~3,000 元** |
| RDS 2核4G | ~320 元 | ~200 元 | **~1,440 元** |

### 7.2 CDN 流量包（必做）

| 流量包类型 | 价格 | 单价 | 节省 |
|-----------|------|------|------|
| 按量付费 | - | 0.24 元/GB | 基准 |
| 1TB 月流量包 | ~180 元/月 | ~0.18 元/GB | **省 25%** |
| 10TB 年流量包 | ~1,500 元/年 | ~0.15 元/GB | **省 37%** |

> 预估月均 700GB，购买 1TB 月流量包最灵活，用不完可累积。

### 7.3 OSS 存储分级（必做）

已纳入生命周期策略，自动执行，无需维护。10 个月周期内，平均存储成本约为标准存储的 40%。

### 7.4 双码率动态降级（可选）

```
存储策略：
  最近 1 个月：1080p + 720p（双码率，活跃期）
  1-3 个月：仅 720p（省 50% 空间）
  3-10 个月：仅 720p + 归档（省 70%+ 费用）
  10 个月后：自动删除
```

### 7.5 流量控制

- 小程序端默认 720p，WiFi 下手动切换 1080p
- 教师点评场景强制 1080p（细节不可丢）
- CDN 防盗链 + URL 鉴权，防止恶意刷流量
- 考试前预热热门视频，减少回源成本

## 八、双码率播放器实现（小程序端）

```javascript
// 小程序端自动切换清晰度
Page({
  data: {
    videoUrl: '',
    currentQuality: '1080p',
    qualityOptions: [
      { label: '高清 1080p', value: '1080p' },
      { label: '流畅 720p', value: '720p' }
    ]
  },

  onLoad() {
    // 根据网络自动选择默认清晰度
    wx.getNetworkType({
      success: (res) => {
        const isWifi = res.networkType === 'wifi';
        this.setData({
          currentQuality: isWifi ? '1080p' : '720p'
        });
        this.loadVideo();
      }
    });
  },

  loadVideo() {
    const { currentQuality } = this.data;
    wx.request({
      url: `https://api.xxx.com/video/xxx?quality=${currentQuality}`,
      success: (res) => {
        this.setData({ videoUrl: res.data.url });
      }
    });
  },

  switchQuality(e) {
    const quality = e.currentTarget.dataset.quality;
    this.setData({ currentQuality: quality }, () => {
      this.loadVideo();
    });
  }
});
```

## 九、与方案一的对比

| 对比项 | 方案一（OSS 直连）| 方案二（CDN 加速）|
|--------|------------------|------------------|
| 年费用（标准估）| ~1.3 ~ 1.5 万 | **~1.1 ~ 1.3 万** |
| ECS 带宽 | 5M | 3M（播放全走 CDN）|
| 播放速度 | 够快（同地域 <100ms）| **更快（全国 <50ms）** |
| 并发播放 | ~300 人同时 | **~500+ 人同时** |
| 流量成本 | 较高（OSS 外网流出贵）| **更低（CDN 流量单价低）** |
| 带宽压力 | 全压在 ECS 5M 上 | **ECS 只收上传，播放走 CDN** |
| 配置复杂度 | 简单 | 中等（需配 CDN 缓存、防盗链）|
| 推荐场景 | 学生集中在广州/广东 | **学生分布多地 / 网课需求 / 追求体验** |

> 广州增城艺考培训班，学生大概率集中在本地。方案一 OSS 直连够用且简单。如果后续有外地分校、网课直播回放需求，或追求极致播放体验，方案二更优。
