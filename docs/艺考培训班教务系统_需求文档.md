# 艺考培训班教务系统 — 项目需求文档

> **文档用途**：辅助 AI 编程工具（Cursor 等）进行代码生成与功能开发，后续可在此基础上补充细节与注意事项。
>
> **系统定位**：
> - **WEB 端**：面向教师 / 教务人员，适合大屏数据查看与批量操作
> - **微信小程序端**：轻量化移动应用，面向学生与家长，强调即时触达与便捷查看

---

## 一、技术栈建议（待确认）

| 层级 | 建议方案 | 备注 |
|------|----------|------|
| 前端 WEB | Vue 3 + Vite + Element Plus | 可替换为 React + Ant Design |
| 小程序 | 微信原生 / Taro / uni-app | 支持多端时选 Taro / uni-app |
| 后端 | FastAPI (Python) | RESTful API |
| 数据库 | MySQL + Redis | 结构化数据 + 缓存 |
| 文件存储 | 腾讯云 COS / 本地 MinIO | 视频资源存储 |
| 认证 | JWT + 微信 OAuth | 小程序用微信授权 |

## 二、角色体系

系统共包含 **四类角色**，权限由后台配置：

| 角色 | 英文标识 | 使用端 | 说明 |
|------|----------|--------|------|
| 教务 | `admin` | WEB 端 | 最高管理权限 |
| 教师 | `teacher` | WEB 端 + 小程序 | 班级教学管理 |
| 学生 | `student` | 小程序 | 查看成绩、视频、请假 |
| 家长 | `parent` | 小程序 | 查看子女信息 |

## 三、WEB 端功能模块

### 3.1 数据仪表盘

**模块说明**：首页概览，根据登录角色展示不同维度的统计数据。

#### 功能点

| 角色 | 展示数据 |
|------|----------|
| 教务 | 教师人数、学生人数、视频资源数、艺考咨询信息数 |
| 教师 | 学生人数、家长人数、视频资源数、艺考咨询信息数 |

#### 开发要点
- 根据登录 Token 中的角色字段，动态渲染不同统计卡片
- 数据来源：各业务表聚合查询，建议使用缓存降低数据库压力
- 可使用 ECharts / Chart.js 展示趋势图（后续扩展）

---

### 3.2 人员信息管理

**模块说明**：管理系统内所有用户的基础信息，支持批量操作。

#### 子模块：人员管理

| 功能 | 描述 |
|------|------|
| 人员基本信息编辑 | 修改姓名、性别、联系方式、头像等基础字段 |
| 账号启用/禁用 | 控制用户是否可以登录系统 |
| 年级/班级分组管理 | 将学生归属到指定年级和班级 |

#### 开发要点
- 用户表需包含：`user_id`、`role`、`name`、`gender`、`avatar`、`phone`、`is_active`、`grade_id`、`class_id`
- 年级/班级为独立字典表，与用户多对多关联（学生）或一对多（教师带班）
- 账号启用/禁用需同步使 JWT Token 失效（黑名单机制或短期 Token）

---

### 3.3 视频管理

**模块说明**：管理学生上传的视频资源，包含审核与评论功能。

#### 子模块：视频审核

| 功能 | 描述 |
|------|------|
| 查看视频 | 列表展示所有视频，支持按学生/班级/上传时间筛选 |
| 编辑视频 | 修改视频标题、标签、所属科目 |
| 删除视频 | 物理/软删除视频资源 |

#### 子模块：教学评论

| 功能 | 描述 |
|------|------|
| 点评学生上传视频 | 教师对指定视频提交文字评语，关联到视频记录 |

#### 开发要点
- 视频存储建议使用对象存储（COS/MinIO），数据库只存储视频 URL 与元数据
- 审核状态字段：`pending`（待审核）/ `approved`（通过）/ `rejected`（驳回）
- 评论支持富文本或纯文本（待确认）
- 视频标签来自字典管理模块

---

### 3.4 成绩管理

**模块说明**：录入和维护学生的学科成绩。

#### 子模块：成绩录入

| 功能 | 描述 |
|------|------|
| 批量导入 | 通过 Excel 模板批量上传成绩数据 |
| 编辑/修改成绩 | 单条记录的增删改操作 |

#### 开发要点
- 支持 `.xlsx` 文件上传，使用 `openpyxl` 或 `pandas` 解析
- 成绩科目维度（来自图片）：**练声、鉴赏、乐理、写作、理论、大考视唱、大考声乐、大考器乐、大考总成、大考排名**
  > ⚠️ 待确认：大考系列是否与日常科目分开独立存储，还是同一张表加类型字段区分
- 成绩表关键字段：`score_id`、`student_id`、`subject`、`score`、`exam_date`、`teacher_id`、`remark`

---

### 3.5 字典管理

**模块说明**：维护系统基础数据字典，供其他模块调用。

#### 子模块：系统字典管理

| 功能 | 描述 |
|------|------|
| 视频标签管理 | 增删改查视频分类标签 |
| 年级/班级管理 | 维护年级与班级的层级关系 |

#### 开发要点
- 字典表设计：`dict_type`（类型标识）+ `dict_value`（值）+ `dict_label`（显示名）+ `sort`（排序）
- 年级/班级建议独立两张表，班级通过外键关联年级

---

### 3.6 权限配置

**模块说明**：管理系统内角色与功能菜单的访问权限。

#### 子模块：基础权限配置

| 功能 | 描述 |
|------|------|
| 角色权限分配 | 为教务、教师、学生、家长四个角色分配可访问的功能模块 |

#### 开发要点
- 当前阶段暂定四个固定角色：`admin`（教务）、`teacher`（教师）、`student`（学生）、`parent`（家长）
- 权限粒度目前为菜单/功能级别，暂不做字段级权限
- 建议采用 RBAC 模型：`用户 → 角色 → 权限`
- 后端使用中间件/装饰器校验 Token 中的角色

## 五、数据库核心表设计（草图）

```
users               — 用户基础表（含角色、状态、微信 OpenID）
grades              — 年级表
classes             — 班级表（关联年级）
teacher_profiles    — 教师扩展信息
student_profiles    — 学生扩展信息（含班级、宿舍、关联教师）
parent_profiles     — 家长扩展信息
parent_student      — 家长-学生关联表（多对多）
videos              — 视频表（含状态、标签、上传者）
video_comments      — 视频点评表
scores              — 成绩表（含科目、考试类型、日期）
leave_requests      — 请假申请表（含状态流转）
discipline_records  — 违纪记录表
dict_items          — 字典数据表
role_permissions    — 角色权限配置表
```

## 六、待确认事项清单

> 开发前需逐一确认，避免返工：

- [ ] **大考排名** 是否作为独立科目存储，还是从大考总成自动计算？
- [ ] **家长与学生绑定方式**：邀请码、手机号匹配，还是后台手动关联？
- [ ] **请假审核流程**：是否需要多级审批（班主任 → 教务）？是否通知家长？
- [ ] **视频点评**：是否支持富文本？是否支持语音点评？
- [ ] **成绩是否区分学期**：同一学生同一科目是否按学期归档？
- [ ] **通知推送**：是否需要微信订阅消息推送（成绩发布、审批结果等）？
- [ ] **多子女支持**：家长端是否支持绑定多个子女？
- [ ] **视频大小与格式限制**：上传的约束条件？
- [ ] **数据权限隔离**：教师是否只能看到自己班级的数据，还是全部学生？

## 七、开发优先级建议

| 阶段 | 模块 | 说明 |
|------|------|------|
| P0 核心 | 用户认证（登录/注册/权限）| 所有功能的基础 |

| P1 主功能 | 成绩录入 + 成绩查询（WEB + 小程序）| 核心业务 |

| P2 次功能 | 数据仪表盘 | 数据展示 |
| P3 扩展 | 权限配置 UI | 初期可硬编码 |
